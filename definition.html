Workload scenario definition file

Type: object

Testbed workloads are defined in YAML files containing information about the desired network configurations and computational workloads to be deployed. We call these "workload scenarios" or simply "scenarios". This document describes the schema these files must follow, and will also serve as a guideline for the design and construction of the software necessary to parse these configurations.
Example:

backend-workload-configs:
  cleave-backend:
    image: cleave-controller:latest
client-workload-configs:
  cleave-client:
    image: cleave-plant:latest
radio-base-configs:
  wifi-ap:
    config-img: wifiap:latest
    params:
      channel: auto
      ssid: testbed_wifi
radio-client-configs:
  wifi-native:
    type: native
version: 1
workloads:
  wl1:
    client-device: generic
    client-workload: cleave-client
    cloudlet-workload: cleave-backend
    network:
      base: wifi-ap
      client: wifi-native
    num-clients: 3

root version
Type: integer Default: 1

Indicates which version of the specification this definition file adheres to. For now, should be set explicitly to 1.

root workloads
Type: object

Dictionary which contains definitions for workloads running in parallel. Each key-value pair corresponds to a workload name and its definition.

Each additional property must conform to the following schema

root workloads workload.json
Type: object

High level definition of a workload

root workloads WorkloadDefinition client-device
Type: string Default: "generic"

Identifier to match against available client device types.
Example:

client-device: generic

root workloads WorkloadDefinition num-clients
Type: integer

Number of clients to instantiate for this workload. A negative value indicates to use all remaining available clients; having more than one workload with a negative value assigned to this property in a scenario will cause an error to be raised.
Examples:

num-clients: 12

num-clients: -1

root workloads WorkloadDefinition client-workload
Type: string

Identifier corresponding to the client workload configuration to use.
Example:

client-workload: base-client-workload-1

root workloads WorkloadDefinition backend-workload
Type: string

Identifier corresponding to the backend workload configuration to use.
Example:

backend-workload: base-backend-workload-1

root workloads WorkloadDefinition network
Type: object

Network configuration for this workload.
Example:

network:
  base: wifi-ap-base
  client: wifi-client-native
